# 项目约定

## 代码风格

- 禁止使用 `any` 类型，必须使用明确的类型定义
- 启用严格的 TypeScript 配置
- 所有变量和函数都需要正确的类型标注
- 组件使用 PascalCase 命名，函数和变量使用 camelCase 命名

## Git 工作流

- 提交信息简洁清晰，前缀约定：`feat:` / `fix:` / `docs:` / `refactor:` / `style:` / `test:`
- 主分支：`main`，开发完成后通过 Pull Request 合并
- 强制推送主分支前需要确认

## 安全规则

- 不要修改 `.env` 文件，避免泄露敏感信息
- 不要在代码中硬编码敏感信息
- 用户密码使用 bcrypt 哈希存储
- 认证 token 需要正确验证，避免未授权访问
- API 对用户输入进行适当验证，避免 SQL 注入、XSS 等常见安全问题

## 测试要求

- 保留现有测试文件不要删除
- 新增功能应考虑添加相应的测试
- 提交代码前必须通过 `pnpm lint` 和 `pnpm typecheck`
