# Phase 1 完善：情绪日记核心模块

## 提案概述

补全 Phase 1 未交付的核心功能——情绪日记模块。当前代码库已有用户体系、认证、任务、笔记等基础能力，但缺失 PRD 中定义的产品灵魂：**事件+情绪+需求三要素结构化记录、情绪健康检测、AI 情绪识别**。同时新增 AI 服务层（智谱 AI + DeepSeek V4）和设置管理。

## 目标

- [ ] 新增 3 张数据库表：emotion_records、emotion_questions、emotion_daily_checks
- [ ] 后端情绪日记 CRUD API
- [ ] 后端情绪健康检测 API（每日出题、答题、评分、阈值提醒）
- [ ] 后端 AI 服务层（智谱 AI + DeepSeek V4 双模型支持）
- [ ] 后端 AI 情绪识别 API（用户点击触发，返回情绪类型+强度）
- [ ] 后端设置管理 API（AI 模型配置、偏好存储）
- [ ] 前端 5 个情绪相关页面：列表、新建、编辑、检测、分析
- [ ] 前端设置页增加 AI 模型配置区域
- [ ] Dashboard 接入真实 API，增加情绪状态卡片
- [ ] 导航菜单增加情绪日记入口
- [ ] 共享类型包增加情绪相关类型定义
- [ ] 种子数据导入（18 道情绪检测题）

## 非目标（后续阶段实现）

- 任务关联情绪记录（Phase 3 R016）
- 多周期分析+决策辅助（Phase 4）
- 客户端加密（后续迭代）
- 定时任务主动推送（等微信接入后实现）
- 微信接入（后续迭代）
- 对话模块功能化（暂无后端，保持不动）

## 背景

Phase 1 提案中规划的 6 个核心目标只完成了基础设施部分（用户体系+认证+管理后台）。开发过程中因样式调试需要，提前实现了任务和笔记模块，但情绪日记——作为产品差异化核心的模块——尚未落地。本次变更回归 PRD 路线，补全情绪日记全部功能。

## 范围

### In Scope

- 数据库：emotion_records、emotion_questions、emotion_daily_checks 三张表
- 后端：情绪日记 API、情绪检测 API、AI 服务层（双模型）、设置管理 API
- 前端：情绪日记 5 个页面、设置页 AI 配置、Dashboard 情绪卡片、导航更新
- 共享类型：情绪相关类型定义

### Out of Scope

- 任务-情绪关联（linkedEmotionId，Phase 3）
- 多周期分析页面和决策辅助（Phase 4）
- 客户端加密
- 对话模块改动
- 日历模块改动
- 移动端底部导航（BottomNav，可单独迭代）

## 风险评估

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| LLM API 不可用/超时 | 中 | 前端 AI 识别按钮为可选操作，失败不阻塞记录保存；后端超时 30s 降级返回 |
| 双模型切换复杂度 | 低 | 统一 AI 服务层抽象，模型切换对上层透明 |
| 种子数据需要手动导入 | 低 | 提供 SQL 文件 + drizzle seed 脚本 |
| Dashboard 重构影响现有功能 | 低 | 保持现有卡片结构，替换数据源为 API 调用 |
